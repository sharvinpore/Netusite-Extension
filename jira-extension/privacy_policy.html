BoardSide — Privacy Policy

Effective date: April 27, 2026

Short version: This extension stores your Jira credentials (domain, email, and API token) exclusively in your local browser using Chrome's built-in storage. No data is ever transmitted to any server other than your own Atlassian instance.

What data we collect

The extension stores the following data locally on your device:

  • Jira domain — e.g. yourcompany.atlassian.net
  • Email address — used to authenticate with the Jira API
  • Atlassian API token — used to authenticate API requests; stored in obfuscated form
  • Cached task data — project and issue information fetched from your Jira instance, stored locally to support offline viewing
  • UI preferences — pinned projects and last sync time

How your data is stored

All data is stored exclusively in chrome.storage.local, which is sandboxed to your Chrome profile on your device. Your API token is stored in obfuscated (non-plaintext) form. No data is written to any external database, server, or cloud service operated by this extension.

How your data is used

Your credentials are used solely to make authenticated requests to the Atlassian REST API at the domain you provide. Specifically:

  • Fetch Jira issues assigned to you or unassigned within your projects
  • Fetch issue details, comments, and worklogs
  • Submit worklog entries on your behalf

No requests are made to any server other than https://<your-domain>.atlassian.net.

Data sharing

We do not share, sell, rent, or transmit your data to any third party. The extension has no analytics, telemetry, or remote logging of any kind.

Data retention & deletion

All stored data can be removed at any time by opening the extension's Settings panel and clearing your credentials, or by uninstalling the extension. Uninstalling Chrome or clearing extension data in Chrome settings will also erase all stored data.

Permissions used

  • storage — to save credentials and cached tasks locally
  • tabs — to open the Jira issue URL or API token page in a new tab
  • host_permissions (*.atlassian.net) — to make authenticated API calls to your Jira instance from the background service worker, bypassing browser CORS restrictions

Changes to this policy

If this policy changes materially, the effective date above will be updated. Continued use of the extension after any update constitutes acceptance of the revised policy.

Questions? Contact the developer via the Chrome Web Store listing page.
